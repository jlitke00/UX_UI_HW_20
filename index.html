Taskify Mobile App Design

From chaos to control, conquer your tasks with taskify.

Problem

Many people struggle with reminders of tasks and keeping them organized in one place. We designed this app to bring all of their tasks into one place in a simple format.

My Roles & Responsibilities

  • We worked as a group of 3.
  • We all shared a lot of the responsibilities.
  • My responsibility in prototyping was the creation of the onboarding pages and the home page. I also did a lot of user testing.

Goal

Our goal was to simplify the experience user have for making and managing tasks. We wanted to showcase our ability to manage tasks by location with pop-up reminders triggered by location.

Timeline

Our project took 3 weeks to complete.

My Roles

Some of what I did in this project were some of the user interviews and a lot of user
testing. Another one of my main responsibilities for this project was creating all of the
onboarding frames and the home page, along with most of the sub pages within the
home page.

User Interviews

For this project we conducted 6 user interviews that centered around how people plan

Problem Statement

After conducting our research, we have observed that busy individuals often struggle to
stay on top of tasks, leading to inefficiency and overwhelm. How might we assist these
forgetful individuals in planning and adhering to their tasks, thereby relieving stress and
helping them achieve their goals?

User Journey Map

We created this user Journey map to come up with how a person might go about finding
and using the app and why they did. This journey map gave us an opportunity to find
things to include in the app.

Sketches To Wireframes

We drew up some sketches and then from there we created some wireframes which
reiterated on the sketches. From there we made the wireframes interactive to run some
user testing.

Usability Test Results

Some of the main takeaways from the user testing were:

  1. The confirm task button needed to be moved to bottom.
  2. Allow option to get directions to the task from the current location.
  3. Change the appearance of the confirm task button.
  4. Due by button needs a calendar and time selection option.

Interactive Prototype

We took the results from the usability testing and turned the wireframes into a hi-fi
clickable prototype. We finalized all of the menu options and buttons and figured out
the best transitions to use for each interaction.

Final Thoughts

Reflection On Project:

We’ve created an app that helps busy individuals effectively plan and implement task reminders, aiming to streamline users’ schedules and reduce their overall stress. Working as a group enhanced the app development experience. We overcame obstacles and gained a broader understanding of users’ needs by sharing different perspectives.

Future Iterations:

  • An A.I. feature that remembers users' recurring tasks and recommends ways to schedule out each week.
  • Voice command connectivity with Siri.
  • The ability to view tasks by week from the calendar.
